# Code of conduct

KODE OS is a small project run by a small team. We want it to stay a place where contributors of any experience level can show up, do good work, and feel respected.

## In short

- Be kind.
- Assume good intent, especially across language and timezone gaps.
- Critique code, not people.
- If you wouldn't say it to a coworker on day one, don't post it.

## Unacceptable

- Harassment, doxxing, slurs, or sexualised language directed at anyone.
- Personal attacks, insults, dismissive "git gud" replies.
- Sustained off-topic disruption of issues or discussions.
- Sharing someone's private information without consent.

## Enforcement

If you witness or experience behaviour that breaks the rules above, email `conduct@kode-nas.com` *(live with the first public alpha — until then, message the maintainers via GitHub)*.

We will:

1. Acknowledge receipt within 72 hours.
2. Talk to the people involved privately.
3. Take action proportional to the severity — a private nudge, a public warning, a temporary ban, or a permanent ban.

Maintainers are not above this code. Reports about maintainer conduct go to the same address; another maintainer handles them.

## Scope

This applies in every project space — GitHub issues, pull requests, discussions, the project's chat channels, and direct messages clearly related to the project.

It also covers public statements made in a way that could reasonably be associated with the project (e.g. "as a KODE OS maintainer, I think…").
